## 4. 机械安装记录

### 4.1 底座固定

底座通过两个 C 型夹具固定在桌板边缘。

实际固定方式：

1. 机械臂底座完整放在桌面上，不让底座悬空。
2. 两个夹具分别布置在底座左右两侧。
3. 夹具上压头压住底座伸出的金属安装平面。
4. 夹具下压头顶住桌板底面。
5. 两侧交替拧紧，避免单侧先完全锁死导致底座偏斜。
6. 固定后用手轻推底座，确认不滑动、不翘起。
7. 检查夹具不遮挡底座正面的电源、网口和 USB 接口。

注意事项：

- 不夹圆形关节。
- 不压塑料外壳、铭牌或接口区域。
- 不只使用一个夹具。
- 不让夹具仅压住很小的边缘。
- 需要长期高强度运行时，可考虑通过底座安装孔固定到独立金属板或厚木板，再固定安装板。

### 4.2 夹爪安装

夹爪已安装完成。

依据 A3 手册，末端法兰具有 3 个 M4 螺纹孔，夹爪使用 3 颗 M4 螺丝固定。现场安装要求：

1. 安装过程保持机械臂断电。
2. 夹爪开合方向与法兰安装方向对齐。
3. 三颗螺丝均匀拧紧。
4. 夹爪线连接夹爪电机与机械臂末端接口。
5. 确保连接线保留足够余量，机械臂运动时不被拉紧。
6. 安装后进行开合测试，结果正常。

### 4.3 原装相机

原装相机组件已安装到机械臂上，并通过 USB 连接到 Ubuntu 20.04 虚拟机。当前系统已能识别该相机为标准 UVC 摄像头。

Ubuntu 端识别信息：

```text
lsusb:
Bus 003 Device 005: ID 05a3:9230 ARC International Camera

v4l2-ctl --list-devices:
USB 2.0 Camera: HD USB Camera (usb-0000:03:00.0-2):
    /dev/video0
    /dev/video1
```

设备节点：

```text
/dev/video0: Video Capture, Streaming, Extended Pix Format
/dev/video1: Metadata Capture, Streaming, Extended Pix Format
```

当前判断：

- `/dev/video0` 是主图像采集节点。
- `/dev/video1` 是 UVC payload header metadata 节点，不作为普通图像输入使用。
- 驱动为 `uvcvideo`，驱动版本显示为 `5.15.178`。
- 默认图像格式为 `1920x1080 MJPG 30 fps`。
- 相机控制项支持亮度、对比度、饱和度、白平衡、gamma、gain、工频、防背光、自动曝光等 UVC 参数。
- VMware USB 透传可用，已通过 `guvcview` 完成实时预览验证。
- `guvcview` 下 30 fps 预览流畅。
- 当前画面方向与常规使用方向上下颠倒；暂不方便物理调整相机方向，后续在软件采集或 ROS 图像链路中做翻转校正。
- 已新增 ROS Noetic 视觉包 `carm_a3_vision`，用于直接通过 V4L2 读取 `/dev/video0` 并发布 ROS 图像话题。
- `carm_a3_vision` 默认使用 `640x480 YUYV 30 fps` 输入，转换为 `rgb8` 后发布。
- `carm_a3_vision` 默认启用 `rotate_180: true`，用于软件修正当前相机安装方向。

支持的主要格式：

| 像素格式 | 分辨率 | 帧率 |
|---|---|---:|
| MJPG | 320x240 | 30 fps |
| MJPG | 640x480 | 30 fps |
| MJPG | 800x600 | 30 fps |
| MJPG | 1024x768 | 30 fps |
| MJPG | 1280x720 | 30 fps |
| MJPG | 1280x1024 | 30 fps |
| MJPG | 1920x1080 | 30 fps |
| YUYV | 320x240 | 30 fps |
| YUYV | 640x480 | 30 fps |
| YUYV | 800x600 | 21 fps |
| YUYV | 1024x768 | 6 fps |
| YUYV | 1280x720 | 9 fps |
| YUYV | 1280x1024 | 6 fps |
| YUYV | 1920x1080 | 6 fps |

后续使用前需确认：

- 相机具体型号；
- 安装支架朝向；
- 相机与夹爪的机械干涉；
- 手眼标定方式；
- 机械臂负载、TCP 和重心参数是否需要更新。
- ROS 中使用 `usb_cam`、`v4l2_camera` 或 OpenCV 直接采集的具体方案。
- 内参标定和图像话题命名。
- 图像方向软件校正方式，例如垂直翻转或旋转 180 度。

## 5. 电源与急停

### 5.1 供电

本机铭牌明确为 `24V DC` 版本，应使用配套 24V 电源，不得接入 48V 电源。

A3 底座主要接口包括：

- UPS 接口；
- 状态指示灯；
- 主电源接口；
- RJ45 千兆网口；
- USB 3.0 接口。

通电前检查：

- 电源输出规格与本机 24V 铭牌一致；
- 线缆无松脱、破损和短路风险；
- 正负极和插头方向正确；
- 急停处于释放状态；
- 底座和夹爪均已可靠固定；
- 工作区内无人和障碍物；
- 网线、夹爪线和其他外接线不会被运动部件拉扯。

### 5.2 正常停止与断电约定

目前未发现独立的软件“系统关机”入口，实验室暂采用以下正常断电流程：

1. 停止正在运行的程序或动作。
2. 将机械臂移动到安全、稳定、无碰撞风险的姿态。
3. 在网页上位机中下使能。
4. 确认末端和前臂不会在失力后撞击桌面或设备。
5. 必要时由一人托住前臂或末端，手指避开关节缝隙。
6. 从交流输入侧关闭插线板或切断电源。
7. 等待指示灯熄灭后再离开。

不建议在带电状态下直接插拔底座直流电源插头。

### 5.3 急停按钮使用

急停用于机械臂失控、即将碰撞、夹具松动、线缆拉扯、出现异常声音或其他危险情况，不作为日常开关使用。

使用方式：

- 紧急情况直接按下红色蘑菇头，按钮会锁定。
- 按下后机械臂会停止供能。
- 解除急停前先排除危险原因。
- 建议先切断交流输入，再按按钮标识方向旋转，使按钮弹起。
- 重新上电后，通过网页上位机检查状态并按需要复位。

安全提醒：

- A3 关节无抱闸保护。
- 当前未发现独立 UPS，按下急停或突然断电后，机械臂可能失力下坠。
- 紧急情况下优先保证人员安全，不把手伸入关节或机械臂运动路径。
- 正式实验前应确保急停按钮位于操作者可立即触及的位置。

## 7. 首次网络配置过程

### 7.1 出厂网络参数

A3 出厂默认参数：

```text
机械臂 IP：10.42.0.101
子网掩码：255.255.255.0
默认网关：10.42.0.1
网页地址：http://10.42.0.101
```

### 7.2 最终采用的首次连接方法

为避免影响 Windows 的 Wi-Fi 上网，最终采用“机械臂直连电脑有线网口”的方式进行首次改址。

接线：

```text
机械臂 --网线-- Windows 电脑有线网口
Windows 电脑继续通过 WLAN 上网
```

电脑有线网卡临时设置：

```text
以太网：
IP 地址：10.42.0.201
子网掩码：255.255.255.0
默认网关：留空
DNS：留空
```

测试命令：

```bat
ping 10.42.0.101
```

网页访问：

```text
http://10.42.0.101
```

注意必须使用 `http://`，不是 `https://`。

### 7.3 修改机械臂地址

在网页上位机的设备/IP 设置中，将机械臂修改为：

```text
设备 IP：192.168.31.60
子网掩码：255.255.255.0
默认网关：192.168.31.1
```

修改设备 IP 后，旧网页地址失效属于正常现象。

随后同步将：

```text
控制器 IP：192.168.31.60
后端服务器 IP：192.168.31.60
```

设备 IP、控制器 IP 和后端服务器 IP 应保持一致，否则可能出现网页能打开但机械臂显示未连接的情况。

改址后：

1. 将机械臂网线从电脑拔下。
2. 接回路由器 LAN 口。
3. 浏览器访问 `http://192.168.31.60`。
4. 确认网页上位机能显示关节实时数据。
5. 将电脑有线网卡 IPv4 恢复为自动获取。

## 8. 网络配置过程中遇到的问题

### 8.1 不要用本次尝试过的命令给 WLAN 添加第二 IP

曾尝试执行：

```bat
netsh interface ipv4 add address name="WLAN" address=10.42.0.201 mask=255.255.255.0 skipassource=true store=active
```

在当前 Windows 版本和 WLAN 配置下，该操作没有保留原 DHCP 地址，而是将 WLAN 改成仅使用 `10.42.0.201`，导致：

- 原 `192.168.31.10` 地址消失；
- 默认网关消失；
- DNS 消失；
- 电脑断网；
- 随后出现 `169.254.x.x` 自动配置地址。

因此，本实验环境中不要使用上述命令修改正在负责上网的 WLAN。

### 8.2 WLAN 恢复命令

管理员命令提示符执行：

```bat
netsh interface ipv4 set address name="WLAN" source=dhcp
netsh interface ipv4 set dnsservers name="WLAN" source=dhcp
```

恢复后重新连接 Wi-Fi，检查：

```bat
ipconfig
```

应重新出现：

- `192.168.31.x` 地址；
- `255.255.255.0` 子网掩码；
- `192.168.31.1` 默认网关。

### 8.3 推荐原则

- WLAN 只负责正常局域网和互联网连接，不为机械臂出厂网段临时改址。
- 首次连接出厂 IP 时，使用独立有线网卡配置 `10.42.0.201/24`。
- 独立有线网卡不配置默认网关和 DNS，避免抢占 WLAN 路由。
- 机械臂完成改址后，再接回路由器。

## 11. 开发环境建议

官方软件开发手册推荐环境：

```text
架构：x86_64
操作系统：Ubuntu 20.04.6 LTS
Python：3.8
ROS 1：Noetic
ROS 2：Foxy
```

支持：

- C++
- Python
- ROS 1
- ROS 2
- Web 上位机

Python 接口优先关注官方当前推荐的纯 Python `carm` 包。正式运行运动 Demo 前，建议先完成：

1. 从 Windows 和 Ubuntu 均能 ping 通 `192.168.31.60`。
2. 只建立 SDK 连接，不下发运动命令。
3. 读取设备状态、关节角度和末端位姿。
4. 保存设备固件版本和 SDK 版本。
5. 使用低速、小幅、单关节动作完成 SDK 首次运动测试。
6. 再逐步加入夹爪、相机、视觉和轨迹规划。

建议在 `config/robot.yaml` 中集中保存：

```yaml
robot:
  model: "MAXHUB A3"
  serial_number: "A3YTL2607***-****"
  ip: "192.168.31.60"
  port: null
  rated_payload_kg: 2.0
  supply_voltage_v: 24
```

端口号应以后续 SDK 文档或实际配置为准，未确认前不要自行填写。
